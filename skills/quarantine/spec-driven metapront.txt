# META-PROMPT: Autonomous Agentic Infrastructure Builder

## OBJECTIVE
Create a comprehensive, executable meta-prompt that enables an AI system (single LLM or multi-agent swarm) to autonomously design, build, and deploy complete agentic infrastructure for accomplishing any specified objective in a one-shot manner.

## CORE CONCEPT: THE AGENTIC HARNESS
An "agentic harness" is a complete scaffolding system that provides an LLM or multi-agent system with everything needed to autonomously accomplish complex tasks. It comprises:

- **Structured Knowledge**: Domain expertise encoded in markdown files (skills.md format)
- **Execution Framework**: Tools, scripts, and automation capabilities
- **Context Management**: Data sources, retrieval systems, and information architecture
- **Specification System**: Complete, unambiguous requirements that eliminate all ambiguity
- **Integration Layer**: Connections to tools, plugins, MCPs (Model Context Protocols), and external systems

The agentic harness + comprehensive specification = autonomous capability to build anything.

## YOUR TASK
Design a meta-prompt system that enables an AI to:

1. **Research and discover** the optimal approach for any given objective (using first principles thinking)
2. **Reverse-engineer requirements** by breaking down the objective into atomic components
3. **Build complete infrastructure** including all scaffolding, context, and capabilities needed
4. **Produce executable systems** (not just plans) that work immediately
5. **Scale appropriately** from simple to complex tasks while defaulting to maximum capability

## CRITICAL SUCCESS CRITERIA

Your meta-prompt must enable the AI to:

✓ **Eliminate all ambiguity** through comprehensive specification
✓ **Leave no stone unturned** in requirements analysis and system design
✓ **Work autonomously** with minimal human intervention or iteration
✓ **Handle unknowns** through systematic research and evaluation
✓ **Produce working code and systems**, not just documentation
✓ **Adapt architecture** based on objective complexity (single LLM vs. multi-agent swarm)
✓ **Be immediately executable** when given any objective

## REQUIRED SYSTEM CAPABILITIES

### 1. Intelligent Resource Acquisition
- Identify and gather all necessary data sources
- Determine required expertise domains and encode them in skills.md format

- Evaluate objective complexity to determine optimal topology:

## REQUIRED MULTI-PHASE STRUCTURE

Design your meta-prompt as a sequence of distinct, executable phases. Each phase must include:

- **Clear Objective**: What this phase accomplishes
- **Concrete Deliverables**: Specific outputs with format specifications
- **Success Criteria**: How to verify completion
- **Handoff Protocol**: What gets passed to the next phase

### PHASE 1: Discovery & Requirements Analysis

**OBJECTIVE**: Fully understand the given objective and identify all requirements (known and unknown)

**PROCESS**:
- Analyze the stated objective using first principles thinking
- Break down the objective into atomic components
- Identify explicit and implicit requirements
- Determine complexity level and scope
- Research domain-specific considerations and best practices
- Identify unknowns that require further investigation

**DELIVERABLES**:
- Comprehensive requirements document (no ambiguity)
- Complexity assessment (simple/moderate/complex/enterprise)
- List of unknowns requiring research
- Initial resource requirements

**SUCCESS CRITERIA**: All requirements are documented with zero ambiguity; complexity level is clearly determined

## RESEARCH & EVALUATION FRAMEWORK

For handling unknowns and making architectural decisions, include:

### Research Protocol
When encountering unknowns:
1. Clearly define what is unknown
2. Identify authoritative sources and best practices
3. Research spec-driven development and agentic harness patterns
4. Synthesize findings into actionable knowledge
5. Encode expertise in skills.md format

### Complexity Evaluation Criteria
- **Simple**: Single-step tasks, minimal context, no external integrations
- **Moderate**: Multi-step workflows, moderate context, some tool integration
- **Complex**: Multi-phase projects, extensive context, multiple integrations
- **Enterprise**: Large-scale systems, distributed architecture, comprehensive integration

### Architecture Decision Framework
Choose architecture based on:
- **Single LLM**: Simple to moderate tasks, linear workflows, limited parallelization
- **Multi-Agent Swarm**: Complex tasks requiring specialization, parallel processing, diverse expertise
- **Orchestrator + Specialists**: Enterprise tasks requiring coordination, quality control, and specialized capabilities
